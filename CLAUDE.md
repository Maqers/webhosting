## Architecture Overview

**Maqers** is a React + Vite SPA for an Indian handmade goods marketplace, deployed on Vercel. All routes are client-side; `vercel.json` rewrites everything to `index.html`.

### Data layer — `src/data/catalog.js`

The entire product catalog lives in one static JS file. It exports:
- `categories` — array of category objects (id, name, slug, icon, order, meta.keywords)
- `productsByCategory` — object keyed by category id, each value an array of product objects

**Product shape:** `{ id, categoryId, title, slug, description, price, images[], popular, featured, inStock, tags[], meta: { keywords[], colors[], sizes[], moq, delivery_time, secondaryCategories[], sellerId, sellerCode, personalisation_options[], personalisation_prices[] } }`

Products can appear in multiple categories via `meta.secondaryCategories`. The helper `getAllProducts()` flattens `productsByCategory` and deduplicates.

> **Critical:** Do not shorten or replace catalog.js. The comment at the top of the file ("DO NOT replace this with any shortened version") is a real constraint — short descriptions break SEO and the AI gift assistant.

Category `order` values must be unique integers and must be updated when adding/removing categories to prevent ordering bugs.

### Admin Portal — `src/pages/AdminPortal.jsx`

The admin UI at `/admin` is a self-contained React component that manages the catalog entirely in-browser:
- Authenticates with GitHub credentials (owner/repo/branch/token) stored in component state (never persisted)
- Reads `src/data/catalog.js` from the repo via GitHub Contents API (`ghGet`/`ghPut`)
- Parses the file as text, injects product/category entries, and commits back via `ghPut`
- Product descriptions are AI-generated by calling `/api/generate-description` (an OpenAI Vision serverless function) with an uploaded image
- Seller records are stored in **Supabase** (`sellers_db` table, `seller_docs` storage bucket); the Supabase anon key is embedded in the component and SellerPage

`sanitizeForJS()` strips newlines/tabs from descriptions before writing — this prevents "Unterminated string constant" build errors on Vercel.

### Serverless API — `api/`

Two Vercel serverless functions:
- `api/generate-description.js` — calls **OpenAI GPT-4o-mini** (Vision) to generate product title, description, tags, and keywords from an uploaded image. Requires `OPENAI_API_KEY` env var.
- `api/gift-assistant.js` — calls **OpenAI GPT-4o** with the full product catalog to recommend 5 gifts for a recipient/occasion/budget. Requires `OPENAI_API_KEY`.

Both are POST-only with CORS open to `*`.

### Search — `src/utils/search.js` + `src/data/searchIndex.js`

Client-side search with:
- Levenshtein fuzzy matching for typo tolerance
- Weighted scoring: title (2×) > category (1.5×) > tags (0.8×) > keywords (0.6×) > description (0.5×)
- Query synonym expansion via `searchIndex.js`
- Category-boosted results: if query matches a category name, all products in that category get +80 to their score
- Intent detection maps queries to categories before scoring

### State management

- `CartContext` and `WishlistContext` — React context + `localStorage` (`maqers_cart`, `maqers_wishlist`). Cart supports color/size/personalisation options with per-option price deltas.
- No Redux or external state library.

### Routing

React Router v6. All pages are lazy-loaded (`React.lazy`). `scrollAnimations` are re-initialised on every route change via `useEffect` on `location.pathname`.

Key routes:
| Path | Component |
|------|-----------|
| `/` | Home |
| `/products` | Products (all items, sortable/filterable) |
| `/categories` | Categories |
| `/category/:name` | Categories (filtered) |
| `/product/:slug` | ProductDetail |
| `/by-occasion` | ByOccasion |
| `/by-product` | ByProduct |
| `/maker/:sellerCode` | SellerPage (fetches from Supabase) |
| `/admin` | AdminPortal |
| `/checkout` | Checkout (WhatsApp-based order flow) |

### Styling

Each component and page has a co-located `.css` file. Global overrides and mobile fixes live in `src/styles/` — many are platform-specific patches (iOS Safari, Android scroll, zoom prevention). Import order in `main.jsx` or `App.jsx` determines cascade.

### Product images

Images are served as static files from `public/images/` (referenced as `/images/…` in catalog). Run `npm run compress-images` to batch-compress with sharp before committing new images.

### Sitemap

`scripts/generate-sitemap.js` runs automatically as part of `npm run build` and writes `public/sitemap.xml` with one URL per product slug.

## Description copy rules

When writing or editing product descriptions for catalog.js:
- No em dashes (`—`). Use commas, colons, or rewrite.
- Paragraphs separated with `\n\n` (literal backslash-n, not actual newlines — they're inside JS strings).
- Minimum 180 words.
- End with 4–6 bullet points prefixed with `✨`.
